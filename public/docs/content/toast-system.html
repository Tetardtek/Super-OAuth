🍞 Système de Toast Notifications

Le système de toast de SuperOAuth offre une expérience utilisateur moderne et intuitive pour les notifications en temps réel.

🎯 Démonstration Live

📋 Vue d'ensemble

Le ToastManager est une classe JavaScript qui gère l'affichage de notifications toast dans l'interface utilisateur. Il est automatiquement initialisé au chargement de l'application.

✨ Fonctionnalités Principales

  • 4 types de notifications : Success ✅, Error ❌, Warning ⚠️, Info ℹ️
  • Animations fluides : Entrée/sortie avec transitions CSS
  • Auto-fermeture : Durée personnalisable (défaut 5 secondes)
  • Fermeture manuelle : Clic sur le toast pour le fermer
  • Design moderne : Couleurs, ombres et icônes appropriées
  • Responsive : S'adapte à toutes les tailles d'écran

🚀 Utilisation

Instance Globale

// Instance disponible globalement const Toast = new ToastManager();

Méthodes Principales

// Toast de Succès Toast.success('Connexion réussie !'); Toast.success('Compte créé avec succès !', 3000); // Durée personnalisée // Toast d'Erreur Toast.error('Identifiants incorrects'); Toast.error('Une erreur est survenue', 5000); // Toast d'Avertissement Toast.warning('Les mots de passe ne correspondent pas'); Toast.warning('Attention, action irréversible', 7000); // Toast d'Information Toast.info('Redirection en cours...'); Toast.info('Nouvelle fonctionnalité disponible', 4000);

Personnalisation des Durées

// Durées courtes pour les confirmations rapides Toast.success('Sauvegardé !', 2000); // Durées longues pour les messages importants Toast.error('Erreur critique', 10000); // Toast permanent (ne se ferme pas automatiquement) Toast.info('Message permanent', 0);

Messages Contextuels par Fonctionnalité

🔐 Authentification
// Connexion Toast.success('Connexion réussie ! Bienvenue John'); Toast.error('Identifiants incorrects'); // Inscription Toast.success('Compte créé avec succès !'); Toast.error('Cette adresse email est déjà utilisée'); // Déconnexion Toast.success('Vous avez été déconnecté');
✅ Validation de Formulaires
// Validation en temps réel Toast.warning('Les mots de passe ne correspondent pas'); Toast.error('Tous les champs sont obligatoires');
🔗 Intégration avec les API
async function apiCall() { try { Toast.info('Requête en cours...'); const response = await fetch('/api/endpoint'); const data = await response.json(); if (response.ok) { Toast.success('Opération réussie !'); } else { Toast.error(data.message || 'Erreur inconnue'); } } catch (error) { Toast.error('Erreur de connexion au serveur'); } }

🎯 Bonnes Pratiques

✅ Messages Efficaces

Toast.success('Mot de passe mis à jour'); Toast.error('Format d\'email invalide'); Toast.warning('Session expire dans 5 minutes'); Toast.info('Redirection vers PayPal...');

❌ Messages à Éviter

Toast.error('Erreur'); // Trop vague Toast.success('Ok'); // Pas informatif Toast.info('Quelque chose s\'est passé'); // Inutile

📏 Règles de Rédaction

  • Concis : Messages courts et clairs
  • Contextuels : Spécifiques à l'action effectuée
  • Actionables : Indiquent la prochaine étape si nécessaire
  • Positifs : Encouragent l'utilisateur même en cas d'erreur

🧪 Tests et Debug

Mode Debug

// Activer les logs de debug const Toast = new ToastManager(); Toast.debug = true; // Mode debug // Les toasts loggeront automatiquement : // - Création du toast // - Affichage // - Fermeture // - Erreurs éventuelles

Inspection des Toasts

// Accéder à tous les toasts actifs console.log(Toast.toasts); // Forcer la fermeture de tous les toasts Toast.toasts.forEach(toast => Toast.remove(toast));

📊 Résumé Technique

Le système de toast améliore significativement l'expérience utilisateur en fournissant un feedback immédiat et contextuel pour toutes les actions de l'application. Il remplace avantageusement les anciennes méthodes comme alert() ou les messages statiques dans la page.
